# Documentation Rules

Use estas regras ao alterar documentacao ou comportamento publico.

## Regras

- `README.md` explica setup, requisitos e uso para usuario.
- `ARCHITECTURE_GUIDELINES.MD` explica camadas, contratos e fluxo.
- `PROJECT_GUIDELINES.MD` explica boas praticas gerais.
- `EVOLUTION_RULES.MD` explica regras normativas de evolucao.
- `docs/rules/` contem regras pequenas para consulta rapida.
- Atualize docs quando mudar CLI, formato de saida, dependencia, estrutura ou contrato.
- Nao documente comportamento desejado como se ja existisse.
- Prefira exemplos curtos e executaveis.

## Checklist

- A doc alterada corresponde ao comportamento real?
- O README precisa ser atualizado?
- Alguma rule pequena ficou desatualizada?
- A mudanca altera `projectStructure.json`?
