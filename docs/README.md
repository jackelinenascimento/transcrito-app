# Documentação do projeto Transcrito App

Esta pasta reúne documentos de arquitetura, regras e guias de contribuição para o projeto.

Índice:

- `ARCHITECTURE_GUIDELINES.MD` — orientações arquiteturais e princípios adotados

Como usar

- Leia `ARCHITECTURE_GUIDELINES.MD` antes de propor mudanças arquiteturais.
- Atualize a documentação correspondente quando modificar camadas importantes do sistema.
- Documentos menores podem ser editados diretamente; mudanças maiores devem abrir uma issue descrevendo a motivação.

Como contribuir com a documentação

1. Crie uma branch com um nome claro (ex.: `docs/update-readme-examples`).
2. Faça as alterações e adicione testes/validações de documentação quando aplicável.
3. Abra um Pull Request descrevendo os motivos e referências (issues) relacionados.
